# Settings / MCP 服务器用户场景索引

## 查看配置

- [支持] 作为用户，我可以进入 MCP 服务器页，查看当前已配置的服务器列表。
- [支持] 作为用户，我可以看到服务器名称、类型、描述和入口命令或 URL。
- [支持] 作为用户，我可以看到空态，知道当前还没有配置 MCP 服务器。
- [不支持] 作为用户，我希望看到服务器真实连接状态，而不是只看到配置存在。
- [不支持] 作为用户，我希望看到 MCP 配置文件的完整内容和解析错误。

## 新增与删除

- [支持] 作为用户，我可以新增 stdio 类型 MCP 服务器，填写名称、命令、参数和描述并保存。
- [支持] 作为用户，我可以新增 SSE 或 HTTP 类型 MCP 服务器，填写名称、URL 和描述并保存。
- [支持] 作为用户，我可以删除已配置的 MCP 服务器。
- [不支持] 作为用户，我希望编辑已有服务器，而不是删除后重新添加。
- [不支持] 作为用户，我希望配置环境变量、请求 headers 或鉴权信息。
- [不支持] 作为用户，我希望保存前验证命令、URL、参数是否可用。
- [不支持] 作为用户，我希望导入常见 MCP server 模板，以便快速配置文件、浏览器、数据库等工具。

## 工具与资源

- [支持] 作为用户，当运行契约明确指定已配置的 MCP 服务器名称时，运行时可以读取同一份配置并加载对应 MCP 工具。
- [不支持] 作为用户，我希望在 Settings 里浏览某个 MCP 服务器暴露的工具、资源或 prompts。
- [不支持] 作为用户，我希望在 Settings 里测试调用某个 MCP 工具。
- [不支持] 作为用户，我希望看到 MCP server 的启动日志、stderr、超时和失败原因。
- [不支持] 作为用户，我希望查看每个 MCP 工具最近被哪个 Agent 或任务使用。

## 作用范围

- [不支持] 作为用户，我希望把某个 MCP 服务器分配给部门、Agent、项目或某次运行。
- [不支持] 作为用户，我希望按工作区启用或禁用某个 MCP 服务器。
- [不支持] 作为用户，我希望限制某个 MCP 服务器只能读、不能写，或只能访问指定目录。
- [不支持] 作为用户，我希望完成“添加服务器 -> 验证连通 -> 选择作用范围 -> 运行中实际使用”的完整闭环。
