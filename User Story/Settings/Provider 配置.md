# Settings / Provider 配置用户场景索引

## AI 接入

- [支持] 作为用户，我可以先添加或管理 AI 接入，再选择默认使用的 Provider。
- [支持] 作为用户，我可以为 Claude API、OpenAI API、Gemini API、Grok API 输入 API Key、测试连接并保存接入。
- [支持] 作为用户，我可以编辑、复测、删除已保存的 API Provider 接入。
- [支持] 作为用户，我可以添加多个 OpenAI-compatible 自定义接入，并填写名称、Base URL、API Key、默认模型后保存。
- [支持] 作为用户，我可以把某个已保存的自定义接入设为当前接入。
- [支持] 作为用户，我可以编辑、复测、删除已保存的自定义接入。
- [支持] 作为用户，我可以看到不完整的自定义接入处于待完成状态，并继续编辑或删除它。
- [支持] 作为用户，我可以看到已检测到的本机 Codex Native、Codex MCP、Claude Code 接入。
- [支持] 作为用户，我可以移除或恢复本机 Provider，且只影响当前系统里的可用性。
- [不支持] 作为用户，我希望在 AI 接入里直接完成本机 Codex 或 Claude Code 登录，而不是只看到去本机登录的提示。
- [不支持] 作为用户，我希望在 AI 接入里安装缺失的本机 Provider CLI，以便不离开 Settings 完成接入。
- [不支持] 作为用户，我希望批量导入多个 API Provider 或自定义端点，以便快速迁移已有接入。
- [不支持] 作为用户，我希望给接入添加标签、用途说明或负责人，以便团队环境下快速识别。

## 默认配置

- [支持] 作为用户，我可以从已接入且可用的 Provider 中选择默认 Provider。
- [支持] 作为用户，我可以为默认 Provider 选择或手动输入默认模型。
- [支持] 作为用户，我可以保存默认 Provider 和默认模型配置。
- [支持] 作为用户，我可以在默认 Provider 未完成接入时看到待处理状态。
- [不支持] 作为用户，我希望把未配置、已移除或不完整的 Provider 保存为可用默认 Provider。
- [不支持] 作为用户，我希望为多个自定义接入分别配置独立默认路由，而不是只使用当前激活的自定义接入。
- [不支持] 作为用户，我希望默认配置支持按工作区、部门或用户隔离，以便不同业务空间使用不同默认模型。
- [不支持] 作为用户，我希望保存前看到本次默认配置改动摘要，以便避免误切换全局默认 Provider。

## 高级设置

- [支持] 作为用户，我可以展开高级设置，为支持图像生成的 Provider 配置图像模型。
- [支持] 作为用户，我可以启用或停用某个 Provider 的图像生成能力。
- [支持] 作为用户，我可以发起图像生成测试并看到测试结果。
- [支持] 作为用户，我可以为 Executive、Management、Execution、Utility 分层设置 Provider 和模型。
- [支持] 作为用户，我可以清除按层覆盖，让分层配置回到默认 Provider。
- [不支持] 作为用户，我希望为任意 Provider 配置图像生成，而不是只使用已标记支持图像能力且已接入的 Provider。
- [不支持] 作为用户，我希望按层覆盖能提供推荐模板，例如高质量、低成本、快速响应，以便不用手动选每一层。
- [不支持] 作为用户，我希望为每层配置 token 上限、超时、重试、并发和成本预算。
- [不支持] 作为用户，我希望能预览一个任务最终会命中哪个 Provider、模型和图像配置。

## 接入可用性与故障处理

- [支持] 作为用户，我保存 API Provider 凭证后，可以让该 Provider 进入默认 Provider 选择闭环。
- [支持] 作为用户，我删除 API Provider 后，系统会把相关默认或覆盖配置迁移到备用 Provider。
- [支持] 作为用户，我可以从其他入口跳转到 Provider 配置并聚焦第三方接入。
- [不支持] 作为用户，我希望在未连接配置后端的 web-only 模式下仍能完成 Provider 配置或凭证保存闭环。
- [不支持] 作为用户，我希望看到每个 Provider 的最近失败原因、延迟、成功率和消费情况。
- [不支持] 作为用户，我希望 Provider 连接失败时系统能给出可执行修复步骤，而不是只显示连接失败。
