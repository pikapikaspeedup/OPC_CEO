# Settings / 凭证中心用户场景索引

## 入口

- [支持] 作为用户，我可以通过隐藏的 `api-keys` tab 或外部入口打开凭证中心。
- [不支持] 作为用户，我希望从 Settings 顶部可见 tab 直接进入凭证中心。
- [不支持] 作为用户，我希望 Provider 配置页能清晰标明凭证中心是二级维护入口，以便知道它不是首次接入主路径。

## API Key 管理

- [支持] 作为用户，我可以查看 Anthropic、OpenAI、Gemini、Grok 的 Key 是否已设置。
- [支持] 作为用户，我可以输入新的 Anthropic、OpenAI、Gemini、Grok Key。
- [支持] 作为用户，我可以测试新输入 Key 的连通性。
- [支持] 作为用户，我可以统一保存已输入的 API 凭证。
- [支持] 作为用户，我可以替换已有 Anthropic、OpenAI、Gemini、Grok Key。
- [支持] 作为用户，我可以切换输入框是否显示正在输入的 Key。
- [不支持] 作为用户，我希望查看、复制或显示已保存的真实 Key 内容。
- [不支持] 作为用户，我希望删除或清空已保存的 Key。
- [不支持] 作为用户，我希望复测已保存 Key，而不是必须重新输入 Key 才能测试。
- [不支持] 作为用户，我希望查看每个 Key 的最近测试时间、最近使用时间和失败原因。
- [不支持] 作为用户，我希望对 Key 做命名、备注、轮换计划和过期提醒。

## 本地登录态

- [支持] 作为用户，我可以查看 Codex Native 的本地安装和登录状态。
- [支持] 作为用户，我可以查看 Claude Code 的本地安装和登录状态。
- [不支持] 作为用户，我希望在凭证中心直接完成 Codex Native 登录。
- [不支持] 作为用户，我希望在凭证中心直接完成 Claude Code 登录。
- [不支持] 作为用户，我希望在凭证中心安装缺失的 Codex 或 Claude Code CLI。

## 与 Provider 配置的关系

- [支持] 作为用户，我在凭证中心保存 API Key 后，可以刷新 Provider inventory 供 Provider 配置使用。
- [不支持] 作为用户，我希望在凭证中心管理 OpenAI-compatible 自定义接入。
- [不支持] 作为用户，我希望在凭证中心设置默认 Provider、默认模型或按层覆盖。
- [不支持] 作为用户，我希望凭证中心能显示某个 Key 当前被哪些默认配置、scene 或 layer 使用。
- [不支持] 作为用户，我希望删除凭证前看到影响范围，以便知道哪些 Provider 或任务会失效。
