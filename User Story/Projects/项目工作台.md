# Projects / 项目工作台用户场景索引

## 进入与导航

- [支持] 作为用户，我能从主导航进入 Projects 页面，以便集中查看项目、执行和交付。
- [支持] 作为用户，我能在 Projects 首屏看到项目总览、关键指标、搜索入口和新建项目入口。
- [支持] 作为用户，我能通过 URL 直接进入 Projects 页面，以便刷新或分享后恢复项目工作区。
- [支持] 作为用户，我能通过 URL 直接打开指定项目详情，以便从外部上下文跳转到项目。
- [支持] 作为用户，我能通过 URL 或运行点击进入指定 Run 关联的项目详情，以便聚焦对应执行内容。
- [支持] 作为用户，我能从项目详情返回 Projects 列表，以便继续浏览其他项目。
- [支持] 作为用户，我能从 CEO Office、部门工作台或其他页面进入 Projects，以便承接管理动作。
- [不支持] 作为用户，我希望 URL 直接定位到 Projects 内部的执行流、运行、交付、诊断、checkpoint 或具体 stage tab，以便分享深层状态。
- [不支持] 作为用户，我希望 Projects 内部筛选、聚焦项目、视图模式和展开状态都写入 URL，以便刷新后完全恢复同一浏览状态。

## 项目总览与浏览

- [支持] 作为用户，我能查看进行中项目、阻塞项目、本周完成和待评审等项目指标，以便快速判断整体项目状态。
- [支持] 作为用户，我能在项目树中按部门或工作区查看项目，以便从组织结构进入项目。
- [支持] 作为用户，我能展开查看更多部门，以便访问默认首屏之外的项目集合。
- [支持] 作为用户，我能查看历史项目集合，以便找到已完成、已归档或已取消项目。
- [支持] 作为用户，我能在列表态聚焦某个项目，以便在不离开总览的情况下查看项目摘要。
- [支持] 作为用户，我能在聚焦项目时查看执行工作台摘要、阶段进度、最近运行、阻塞项、下一步、执行概览、工作区和关联运行。
- [支持] 作为用户，我能从浏览态打开项目详情，以便进入完整执行工作台。
- [支持] 作为用户，我能看到空项目状态，并从空状态创建项目或 AI 生成 Pipeline。
- [不支持] 作为用户，我希望自定义项目树排序、字段和显示密度，以便适配不同 review 习惯。
- [不支持] 作为用户，我希望保存个人常用项目视图，以便重复查看固定项目集合。
- [不支持] 作为用户，我希望 Projects 首屏自动推荐我的待办项目、风险项目或最近负责项目，以便减少人工定位。

## 搜索与筛选

- [支持] 作为用户，我能按全部项目、进行中、关注项和历史项目筛选项目。
- [支持] 作为用户，我能按项目名称、目标、状态、模板和工作区关键词搜索项目。
- [支持] 作为用户，我能在顶部搜索和项目树搜索之间保持搜索词同步，以便不同入口一致定位。
- [支持] 作为用户，我能在搜索或筛选无结果时看到空状态提示。
- [不支持] 作为用户，我希望 Projects 搜索触发全量服务端检索，以便不受当前加载范围影响。
- [不支持] 作为用户，我希望组合多个工作区、状态、模板、负责人、创建时间和风险条件做高级筛选。
- [不支持] 作为用户，我希望保存筛选条件、命名视图和个人默认视图。
- [不支持] 作为用户，我希望搜索结果显示命中高亮和排序原因，以便理解为什么匹配。

## 项目创建与维护

- [支持] 作为用户，我能创建项目，填写项目名称、目标、工作区，并可选绑定模板。
- [支持] 作为用户，我能从顶部主按钮、项目树、空状态和快捷操作创建项目。
- [支持] 作为用户，我能编辑项目名称和目标。
- [支持] 作为用户，我能归档项目。
- [支持] 作为用户，我能从归档状态恢复项目，并恢复到系统推断的执行状态。
- [支持] 作为用户，我能删除项目，并在删除前看到确认弹窗。
- [支持] 作为用户，我能在项目操作失败时看到底部错误提示。
- [不支持] 作为用户，我不能在编辑项目时修改工作区、模板、项目类型、技能提示或优先级。
- [不支持] 作为用户，我不能在创建项目弹窗内新建工作区或部门，只能选择已有工作区。
- [不支持] 作为用户，我不能批量选择、批量归档、批量删除或批量派发多个项目。
- [不支持] 作为用户，我不能在项目列表中直接内联编辑项目名称、目标、工作区或模板。
- [不支持] 作为用户，我不能复制项目、克隆项目结构或基于已有项目创建新项目。
- [不支持] 作为用户，我不能手动创建子项目；子项目主要来自 fan-out 或已有父子关联。
- [不支持] 作为用户，我不能撤销删除或从回收站恢复项目。

## 部门树与部门配置

- [不支持] 作为用户，我希望 Projects 左树以部门为一级节点、项目为二级节点，并在同一工作面直接新增部门。
- [不支持] 作为用户，我希望在 Projects 左树中新建部门时选择一个或多个文件夹，并标记主执行目录、执行目录和上下文目录。
- [不支持] 作为用户，我希望点击部门节点后进入完整部门工作台，查看部门概览、项目列表、workspace 绑定、上下文文档、执行策略、Provider、预算和 OKR。
- [不支持] 作为用户，我希望在 Projects 内直接调整部门默认执行 workspace、附加上下文目录和可写范围，以便控制任务实际落在哪个目录运行。
- [不支持] 作为用户，我希望在 Projects 左树里分清“部门节点”和“目录节点”，而不是把部门和工作区混成同一分组概念。

## 项目详情

- [支持] 作为用户，我能在项目详情查看项目名称、状态、创建时间、模板和目标。
- [支持] 作为用户，我能查看最近执行、结果摘要、交付产物数量和关注项。
- [支持] 作为用户，我能查看项目运行次数、已完成运行数、校验结果和关注项摘要。
- [支持] 作为用户，我能在 fan-out 项目详情中查看主项目和关联子项目。
- [支持] 作为用户，我能在主项目和子项目之间切换，以便聚焦不同分支。
- [支持] 作为用户，我能从 fan-out 分支打开关联子项目。
- [支持] 作为用户，我能在项目详情第一层直接看到执行工作台，而不是先进入中间概览。
- [支持] 作为用户，我能在项目详情中切换执行流、运行和交付三个工作台 tab。
- [不支持] 作为用户，我希望在项目详情中看到完整项目成员、负责人、截止时间、业务价值和验收标准。
- [不支持] 作为用户，我希望在项目详情中复制 Project ID、Run ID、Stage ID 或 artifact path。
- [不支持] 作为用户，我希望项目详情支持评论、备注、@同事和协作讨论。

## 部门上下文与 CEO 决策

- [支持] 作为用户，我能在项目绑定部门配置时查看部门名称、类型、描述、技能、工作流绑定、模板数量、Provider 和 OKR 快照。
- [支持] 作为用户，我能从项目快捷操作进入部门设置，以便补齐部门上下文。
- [支持] 作为用户，我能查看 CEO 或 AI 决策记录、原始指令、决策依据、关联部门、模板和决策时间。
- [支持] 作为用户，我能处理需要决策的项目建议，例如转派部门、自动生成并派发模板、创建模板或使用模板。
- [支持] 作为用户，我能在未派发项目中看到待决策或待派发状态，以便选择后续动作。
- [不支持] 作为用户，我希望在 Projects 内完整编辑部门画像、技能、模板和 Provider，而不是跳转到部门设置。
- [不支持] 作为用户，我希望 CEO 决策能形成可分派、可关闭、可审计的任务流。
- [不支持] 作为用户，我希望 AI 决策建议支持用户备注、拒绝原因和重新生成。

## 派发与 Pipeline 生成

- [支持] 作为用户，我能对未派发项目选择模板并派发 Pipeline。
- [支持] 作为用户，我能在派发时填写执行目标。
- [支持] 作为用户，我能在派发前校验模板，并看到有效、错误或警告状态。
- [支持] 作为用户，我能通过 AI Generate 根据目标、约束、技术栈、团队规模和参考模板生成 Pipeline 草稿。
- [支持] 作为用户，我能限制 AI 生成 Pipeline 是否允许 fan-out、loop、gate 和最大阶段数。
- [支持] 作为用户，我能在保存前预览 AI 生成说明、DAG 校验结果、节点列表和风险。
- [支持] 作为用户，我能在存在 critical risk 时阻止保存生成模板。
- [支持] 作为用户，我能恢复本地保存的 Pipeline 生成草稿。
- [支持] 作为用户，我能确认保存 AI 生成的模板，并刷新 Projects 视图。
- [不支持] 作为用户，我不能在 Projects 页面真正完成 pipeline 与 graphPipeline 格式转换保存；当前转换入口缺少完整转换输入和落库闭环。
- [不支持] 作为用户，我不能在确认 AI 模板前可视化编辑节点、边、条件、数据映射或角色配置。
- [不支持] 作为用户，我不能在 Projects 页面完整编辑已有 Pipeline 模板定义。
- [不支持] 作为用户，我不能对模板生成结果做版本对比、多人评审或回滚。

## 执行流

- [支持] 作为用户，我能在执行流中查看 Pipeline 阶段列表、完成进度、当前运行阶段和阶段状态。
- [支持] 作为用户，我能看到待开始、执行中、已完成、失败、阻塞、已取消和已跳过等阶段状态。
- [支持] 作为用户，我能看到阶段重试次数、角色数量、分支数量和执行耗时。
- [支持] 作为用户，我能查看团队角色摘要，包括总角色数、工作中、已完成、排队和待复核。
- [支持] 作为用户，我能点击阶段卡进入阶段详情。
- [支持] 作为用户，我能点击阶段内角色进入角色详情。
- [支持] 作为用户，我能在没有阶段但存在 prompt run 的项目中自动聚焦 prompt run 详情。
- [支持] 作为用户，我能在没有阶段和 prompt run 时看到“暂无执行证据”空状态。
- [不支持] 作为用户，我不能在执行流中拖拽重排阶段或直接编辑阶段定义。
- [不支持] 作为用户，我不能在执行流中批量启动、批量跳过或批量取消多个阶段。
- [不支持] 作为用户，我不能在执行流中为单个阶段设置负责人、截止时间或 SLA。

## 阶段详情与角色详情

- [支持] 作为用户，我能在阶段详情查看阶段状态、耗时、尝试次数、开始时间、完成时间、阶段 ID 和运行 ID。
- [支持] 作为用户，我能查看阶段执行摘要、决策、输出产物、待复核项、阻塞项和错误信息。
- [支持] 作为用户，我能查看阶段任务配置、成功标准和输入产物。
- [支持] 作为用户，我能查看 fan-out 阶段的分支工作面、分支状态、分支耗时、分支 run 和子项目入口。
- [支持] 作为用户，我能对 stale active 阶段轻推继续。
- [支持] 作为用户，我能对失败、阻塞或取消阶段重启角色。
- [支持] 作为用户，我能取消正在运行或阻塞的阶段运行。
- [支持] 作为用户，我能跳过 pending、failed、blocked 或 cancelled 阶段。
- [支持] 作为用户，我能强制完成 running、failed、blocked、cancelled 或 pending 阶段。
- [支持] 作为用户，我能打开阶段关联的 AI 会话。
- [支持] 作为用户，我能对阶段运行触发 AI 诊断。
- [支持] 作为用户，我能在角色详情查看角色状态、轮次、耗时、执行摘要、prompt snapshot、输入读取审计和变更文件。
- [支持] 作为用户，我能在角色详情查看对话步骤抽屉和完整会话入口。
- [不支持] 作为用户，我不能在 Projects 页面为 resume、nudge 或 restart_role 输入自定义 prompt 或指定 roleId。
- [不支持] 作为用户，我不能对单个工具调用、单个角色步骤或单个失败文件执行精细化重试。
- [不支持] 作为用户，我不能在阶段详情中直接编辑 task envelope、成功标准或输入产物。
- [不支持] 作为用户，我不能在角色详情中修改 prompt snapshot 或重新生成角色提示词。

## Gate、Switch、Loop 与 DAG

- [支持] 作为用户，我能在待审批 Gate 出现时从执行流顶部提示定位到对应阶段。
- [支持] 作为用户，我能在 Gate 阶段执行批准或驳回。
- [支持] 作为用户，我能查看 Gate 已批准或已驳回状态、处理时间、处理人和原因字段。
- [支持] 作为用户，我能看到阶段卡上的 Gate 审批状态、loop 轮次和 switch 分支选择。
- [支持] 作为用户，我能在非线性 Pipeline 中切换列表视图和拓扑视图。
- [支持] 作为用户，我能在 DAG 拓扑中查看节点、边、节点状态、节点类型和 fan-out 完成数。
- [支持] 作为用户，我能点击 DAG 节点并联动选择对应阶段。
- [支持] 作为用户，我能在 DAG 加载中看到加载状态，在无 DAG 数据时看到空状态。
- [不支持] 作为用户，我不能在 Gate approve 或 reject 时填写审批原因或审批人。
- [不支持] 作为用户，我不能保证 Gate 批准后下游阶段自动继续触发成完整闭环。
- [不支持] 作为用户，我不能在 DAG 拓扑中直接审批 Gate，需要回到列表和阶段详情处理。
- [不支持] 作为用户，我不能在 DAG 拓扑中拖拽、缩放、搜索、重排、连线、删除或新增节点。
- [不支持] 作为用户，我不能在 DAG 里完整查看 switch 条件、gate outcome 边标签或数据映射。

## Run 与 Prompt Run

- [支持] 作为用户，我能查看项目最近运行和关联运行，并从运行进入项目详情。
- [支持] 作为用户，我能查看独立 Prompt Run 列表、状态、创建时间、耗时、摘要预览、交付物数量和校验标记。
- [支持] 作为用户，我能选择 Prompt Run 查看 Run 详情。
- [支持] 作为用户，我能取消正在运行的 Prompt Run。
- [支持] 作为用户，我能在 Run 详情查看状态、结果摘要、流式输出、文件变更、交付物、校验结论、关注项和技术细节。
- [支持] 作为用户，我能在 Run 详情中查看 AI 对话；有独立 conversation 时打开完整对话，有 transcript 时直接查看消息记录。
- [支持] 作为用户，我能从系统改进 proposal 跳转到关联平台工程项目，以便查看该系统改进是由哪个项目执行的。
- [支持] 作为用户，我能在平台工程项目详情看到结构化的系统改进摘要，而不是直接看到整段 proposal 原文目标。
- [支持] 作为用户，我能从平台工程项目详情重新打开系统改进详情，以便回到前因、执行状态和发布检查视图继续 review。
- [支持] 作为用户，平台工程 Codex CLI runner 的 Run 详情不会再显示无效的“查看 AI 对话”，而是明确提示这是 worktree / diff / 校验 / 发布检查类执行证据。
- [支持] 作为用户，我能在 Run 详情中取消活跃或阻塞 Run。
- [支持] 作为用户，我能在 Run 详情中触发 AI Diagnose。
- [支持] 作为用户，我能在非 executive 模式下对 Run 执行 nudge、retry、restart_role、cancel 和 evaluate。
- [支持] 作为用户，我能在 Run 详情中查看 review queue、blockers、task envelope、result envelope、trace、角色观测和 Supervisor Review Log。
- [不支持] 作为用户，我希望平台工程 Codex CLI runner 项目详情直接内嵌展示 Codex 执行日志、patch 正文、evidence JSON 正文和 worktree diff，而不是通过系统改进详情或 Ops 查看路径与状态。
- [不支持] 作为用户，我不能在 Projects 页面创建 prompt-only project 后直接追加新的 prompt run。
- [不支持] 作为用户，我不能在 Projects 页面稳定看到 agent-run SSE 实时事件；页面主要依赖轮询和刷新。
- [不支持] 作为用户，我不能在 executive 模式下查看 review、envelope、trace 和 chat 技术标签。
- [不支持] 作为用户，我不能从 Projects 页面直接复制、导出或分享完整 Run 详情。

## 运行诊断与 Reconcile

- [支持] 作为用户，我能在运行 tab 查看项目健康状态、摘要和活跃阶段。
- [支持] 作为用户，我能查看每个 stage 的状态、pending 原因、等待上游和异常状态。
- [支持] 作为用户，我能查看 fan-out 分支健康、失败原因和 stale 时间。
- [支持] 作为用户，我能先 dry run 项目 reconcile，再执行 reconcile 修复可调和状态。
- [支持] 作为用户，我能看到 reconcile 结果动作和执行结果。
- [支持] 作为用户，我能在诊断不可用时看到空状态。
- [不支持] 作为用户，我希望诊断结果直接给出业务影响范围、根因归类和一键处置链路。
- [不支持] 作为用户，我希望从诊断项直接跳转到对应 stage、run、role、日志或产物。
- [不支持] 作为用户，我希望诊断自动聚合模型错误、工具错误、权限错误、预算错误和环境错误。
- [不支持] 作为用户，我希望 reconcile 执行前看到完整变更 diff 和影响范围确认。

- [支持] 作为用户，我能在运行 tab 查看项目 Execution Journal。
- [支持] 作为用户，我能查看 gate、loop、switch、checkpoint 等项目控制流事件。
- [支持] 作为用户，我能查看项目最近 audit events。
- [支持] 作为用户，我能查看项目 Policy Compliance。
- [支持] 作为用户，我能看到策略是否允许、是否阻塞，以及资源当前值和限制。
- [支持] 作为用户，我能看到策略违规的 block、pause 和 warn 动作。
- [支持] 作为 API 使用者，我能按 project、workspace 或 template scope 检查 resource policy。
- [支持] 作为 API 使用者，我能创建或覆盖 resource policy。
