## 核心概念

在开始之前，先了解几个反复出现的关键术语：

### 什么是"原子化"？

**原子笔记**是指只包含一个独立主题的短笔记，可以像积木一样被多篇文章复用。与之对应：

| 角色 | 图标 | 含义 | 举例 |
|------|------|------|------|
| **atom（原子）** | 🔹 | 聚焦单个主题的短笔记，可被多篇笔记引用 | `[[HTTP缓存机制]]`、`[[Git Rebase用法]]` |
| **composite（组合）** | 📄 | 通过 `![[嵌入]]` 把多个原子拼装成一篇完整文章 | `[[Web性能优化指南]]`（嵌入了缓存、压缩、CDN 三个原子） |
| **standalone（独立）** | 📝 | 普通笔记，不属于上面两种 | 日常笔记、会议记录 |

> 💡 `![[笔记名]]` 是 Obsidian 的**嵌入语法**，效果是把另一篇笔记的内容直接展示在当前笔记中，类似"引用"。`[[笔记名]]` 则是普通链接，点击后跳转。

### 功能分层：L0 和 L1

本插件分为两个功能层级，**L0 功能完全离线可用，不需要任何外部服务**：

| 层级 | 需要什么 | 包含功能 |
|------|---------|----------|
| **L0（本地）** | 什么都不需要 | 相关笔记、链接建议、Vault 健康报告、断链修复 |
| **L1（AI 增强）** | GitHub Copilot 订阅 | AI 实体提取、原子拆分/合并/升级、浮动工具栏、智能聊天 |

### Gateway 是什么？

Antigravity Gateway 是一个**可选的**独立后端服务（不包含在本插件中），提供 AI 多模型对话和网页搜索能力。如果你不需要在 Obsidian 中搜索互联网或使用 Gateway 的 AI 模型，可以忽略所有 Gateway 相关设置。**绝大多数功能不依赖 Gateway。**

## 安装与配置

### 前置条件

- Obsidian ≥ 1.0.0（桌面版）。移动端暂不支持，因为插件依赖 IndexedDB 和桌面端特有的编辑器 API
- （可选）GitHub Copilot 订阅 — 启用 L1 AI 增强功能。如果你已经在 VS Code 中使用 Copilot，同一个账号直接可用
- （可选）Antigravity Gateway 服务 — 启用 AI 对话和网页搜索。这是一个独立部署的后端服务，需要单独安装和运行

### 安装步骤

1. 获取构建产物 `main.js`、`manifest.json`、`styles.css`（从发布包下载，或自行 `npm run build` 编译）
2. 在你的 Vault 文件夹中找到 `.obsidian/plugins/` 目录，创建 `obsidian-antigravity` 子文件夹
3. 将三个文件复制进去，完整路径形如 `<VaultPath>/.obsidian/plugins/obsidian-antigravity/main.js`
4. 重启 Obsidian
5. 设置 → 第三方插件 → 关闭"安全模式" → 启用 **Google Antigravity Obsidian**

### 首次配置

1. 打开设置页 → 左侧找到 **Antigravity** 选项卡
2. **Gateway URL**（可选）: 默认 `http://localhost:3000`。仅在你已运行 Antigravity Gateway 服务时需要填写，否则留空即可
3. **Copilot 登录**（可选）: 点击 "Login with GitHub Copilot" 按钮 → 浏览器会自动打开 GitHub 设备激活页面 → 输入屏幕上显示的 8 位验证码 → 授权完成后返回 Obsidian，状态会显示"已登录"。登录后所有 L1 AI 功能自动解锁
4. **排除目录**: 配置不参与知识索引的文件夹（默认已排除 `node_modules,.git,dist,build,.obsidian`），一般不需要修改

## Related Notes — 相关笔记

### 打开方式

- 点击侧边栏的 🔗 图标（右侧面板）
- 或使用命令面板（`Ctrl/Cmd + P`）输入 `Open Related Notes Panel`

### 工作原理

每当你切换到一篇笔记，插件会自动分析它与 Vault 中所有其他笔记的关系，给出一个 0–100 的相关度评分。

评分基于 7 个不同维度（称为"信号"），每个维度有不同的权重——权重越高，该维度对最终评分的影响越大：

| 信号 | 权重 | 含义 |
|------|------|------|
| **Embed（嵌入）** | 4.0（最高） | 两篇笔记之间有 `![[嵌入]]` 引用关系 |
| **Link（链接）** | 3.0 | 两篇笔记之间有 `[[链接]]` 引用关系 |
| **Semantic（语义）** | 2.5 | AI 判定两篇笔记讨论的主题相近（需要登录 Copilot） |
| **Tag（标签）** | 2.0 | 两篇笔记共享了相同的 `#标签` |
| **Entity（实体）** | 2.0 | 两篇笔记中出现了相同的关键概念（如人名、产品名、`[[wiki链接]]` 引用的术语） |
| **Token（词频）** | 1.5 | 两篇笔记使用了相似的词汇（基于词频统计） |
| **Title（标题）** | 1.0（最低） | 两篇笔记的标题中有相同的词 |

> 💡 不需要登录 Copilot 也可以使用 Related Notes——语义信号会被跳过，其余 6 路信号仍然工作。

### 界面说明

打开 Related Notes 面板后，你会看到最多 10 篇最相关的笔记，每条包含：

- **分数环**: 笔记左侧的圆环，颜色从绿色（高相关度）渐变到灰色（低相关度）
- **信号标签**: 分数环旁边的彩色小标签，点亮的标签表示该维度对相关度有贡献（如亮起 "link" 和 "tag" 表示两篇笔记既有链接关系又有共同标签）
- **角色徽章**: 显示目标笔记的类型——🔹atom / 📄composite / 📝standalone（参见[核心概念](#核心概念)）
- **Insert Link**: 鼠标悬停在某条结果上时出现的按钮，点击可在当前笔记光标位置插入 `[[链接]]`
- **展开详情**: 点击某条结果可展开查看该笔记被提取到的实体（关键概念）、主题和高频词列表

## LinkSuggester — 链接建议

### 工作方式

编辑笔记时，引擎自动检测当前行文本是否匹配 Vault 中其他笔记的标题。匹配到时在光标位置弹出建议浮窗。

### 操作

| 按键 | 动作 |
|------|------|
| `Tab` / `Enter` | 接受建议，在匹配位置插入 `[[link]]` |
| `Escape` | 忽略建议（本次编辑会话内不再提示该项） |
| `↑` / `↓` | 在多个建议间切换 |

### 特性

- **输入防抖**: 停止输入 0.8 秒后才触发检测，不会打断你的打字节奏
- 不会在 `[[已有链接]]` 或代码块内触发——已经链接过的内容不会重复建议
- 大小写不敏感匹配
- 最多同时显示 5 条建议

## Vault Health — 知识健康报告

### 打开方式

命令面板（`Ctrl/Cmd + P`）: 输入 `Vault Health Report`

### 报告内容

打开后会显示一个完整的面板，从上到下包含以下区域：

1. **健康分 (0–100)**: 综合评估你的 Vault 知识结构质量。分数越高表示笔记之间的连接越紧密、结构越合理
2. **统计卡片**: 一组数字概览——笔记总数、孤立笔记数（没有任何链接的笔记）、过时笔记数、内容重叠数
3. **角色分布**: 显示 atom / composite / standalone 三种笔记类型的比例（参见[核心概念](#核心概念)）
4. **知识网络图**: 一张交互式的网络可视化图（最多显示 100 个节点），用连线展示笔记之间的关系，节点按笔记类型着色
5. **孤立笔记**: 没有被任何笔记链接或引用的笔记列表，点击笔记名可直接打开
6. **断链修复**: 扫描所有指向不存在文件的链接，并提供修复建议（详见[下节](#broken-link-fixer--断链修复)）
7. **内容重叠**: 内容高度相似的笔记对，附带 **AI Merge** 按钮可一键合并（需登录 Copilot）
8. **知识空白**: 某个概念在多篇笔记中被频繁提到，但你还没有为它创建一篇独立笔记——插件会提示你补充
9. **拆分建议**: 如果某篇 standalone 笔记字数 ≥800 且涵盖 ≥3 个主题，会建议你将它拆分为多个原子

### 网络可视化说明

- **蓝色小节点**: atom（原子笔记）
- **琥珀色大节点**: composite（组合笔记）
- **灰色中节点**: standalone（独立笔记）
- **实线**: 表示两篇笔记之间有 `[[链接]]` 关系
- **虚线**: 表示两篇笔记之间有 `![[嵌入]]` 关系
- 点击任意节点可直接打开对应笔记
- 每次打开的布局是固定的，不会随机变化

## Broken Link Fixer — 断链修复

### 检测方式

断链修复集成在 Vault Health 报告中，自动扫描所有笔记中指向不存在文件的 `[[链接]]`。

### 修复建议来源

插件通过三种方式为你找到可能的正确目标：

1. **精确大小写匹配**: 文件名完全相同但大小写不同（如 `[[readme]]` vs `README.md`）
2. **模糊匹配**: 文件名拼写接近（如 `[[JavaScrip]]` → `JavaScript.md`），通过计算文字差异度找到最相似的文件
3. **实体匹配**: 虽然文件名不像，但内容中包含了断链里提到的关键概念

### 操作

在 Vault Health 面板的 "Broken Links" 区域：
- 每个断链显示受影响的文件列表
- 修复建议按相似度排序
- 点击 **Fix** 按钮：自动替换所有源文件中的断链文本，保留别名（`[[old|alias]]` → `[[new|alias]]`）

## AI 增强功能

> ⚠️ 以下功能需要先在设置中登录 GitHub Copilot（参见[首次配置](#首次配置)）

### AI Enrich Notes — 批量智能分析

命令面板: `AI Enrich Notes (Copilot)`

这个命令会让 AI 自动阅读你的笔记，并为每篇笔记提取以下信息：

| 提取内容 | 说明 | 上限 |
|---------|------|------|
| **实体** | 笔记中出现的关键概念、人名、产品名、技术术语等 | 每篇最多 20 个 |
| **主题** | 笔记讨论的核心话题（如"分布式系统"、"HTTP协议"） | 每篇最多 10 个 |
| **关键词** | 重要词汇 | 每篇最多 15 个 |
| **摘要** | 笔记内容的简短概括 | 最多 200 字 |
| **向量嵌入** | 把笔记内容转化为一组数字（用于计算语义相似度） | — |

- 每次运行最多处理 20 篇尚未增强的笔记，可多次运行直到全部完成
- 增强后，Related Notes 的语义信号（Semantic）自动激活，相关度计算更准确
- AI 请求之间有 0.8 秒间隔，避免过快触发 API 限制

### L0 和 L1 的区别

| | 无 Copilot (L0) | 有 Copilot (L1) |
|--|----------------|-----------------|
| Related Notes | ✅ 6 路信号（除语义外） | ✅ 7 路信号全开 |
| LinkSuggester | ✅ | ✅ |
| Vault Health | ✅（除 AI Merge） | ✅ 完整功能 |
| 断链修复 | ✅ | ✅ |
| Inline 续写 | ❌（可选配 Ollama 离线使用） | ✅ |
| 浮动工具栏 | ❌ | ✅ |
| 原子化操作 | ❌ | ✅ |
| 智能聊天 | ❌ | ✅ |

## 原子化操作

> ⚠️ 以下操作需要登录 GitHub Copilot

### 笔记角色

插件会自动判断每篇笔记的"角色"（参见[核心概念](#核心概念)）：

| 角色 | 说明 | 如何被判定 |
|------|------|-----------|
| **atom** 🔹 | 聚焦单一主题、可被复用的知识单元 | 笔记开头的 frontmatter 中写了 `type: atom`，或者该笔记被 2 篇以上笔记用 `![[]]` 嵌入引用 |
| **composite** 📄 | 由多个原子组装成的完整文章 | 笔记中包含 `![[嵌入]]` 引用 |
| **standalone** 📝 | 普通独立笔记 | 不满足以上条件的笔记 |

> 💡 **frontmatter** 是 Obsidian 笔记开头用 `---` 包裹的元数据区域，例如：
> ```yaml
> ---
> type: atom
> tags: [编程, JavaScript]
> ---
> ```

### 两种触发方式

原子化操作支持**手动触发**和**自动触发**两种方式：

#### 方式一：手动触发（命令面板）

对任意笔记，打开命令面板（`Ctrl/Cmd + P`）执行 `AI Atom Split` 或 `AI Atom Upgrade`。适用于你主动想拆分某篇笔记的场景。

#### 方式二：自动触发（标记 `type: knowledge`）

在笔记的 frontmatter 中添加 `type: knowledge`：

```yaml
---
type: knowledge
---
```

这告诉插件"这篇笔记是知识型笔记，请自动分析"。标记后，插件会在以下时机**自动运行 AI 分析**：

1. **编辑后等待** — 你编辑该笔记后，插件启动一个计时器（默认 5 分钟）。在你持续编辑期间计时器会重置，确保不打断写作
2. **离开笔记时** — 如果你切换到其他笔记，插件立即触发分析（不再等待）
3. **分析完成** — AI 分析结果（拆分建议 / 合并候选 / 升级可用性）会自动出现在 **Related Notes 侧边栏**中
4. **你审阅确认** — 点击侧边栏中的建议即可执行，所有操作仍需你确认

> 💡 自动触发需要以下条件同时满足：
> - 在设置中启用了 atomization（`atomizationEnabled`）
> - 已登录 Copilot
> - 笔记 frontmatter 中有 `type: knowledge`
> - 笔记字数 ≥ 800 且包含 ≥ 3 个标题/主题

**建议结果会被缓存**——如果笔记内容没有变化，下次打开不会重复分析，直接展示上次的建议。

### AI Atom Split（拆分）

命令面板: `AI Atom Split (Current Note)`

将一篇涵盖多个主题的长笔记拆分为多个独立的原子笔记：

1. AI 分析当前笔记的标题结构和内容边界，识别哪些段落属于不同主题
2. 弹出拆分预览：列出计划创建的每个原子的标题和对应段落
3. 你确认后，插件自动执行：
   - 为每个主题创建独立的原子文件（自动添加 `type: atom` 到 frontmatter）
   - 将原笔记转为 composite——内容被替换为 `![[原子1]]` `![[原子2]]` 的嵌入引用
   - 如果已存在同名文件，会自动跳过不覆盖

### AI Atom Merge（合并）

在 Vault Health 面板的 "Content Overlaps" 区域，如果两篇笔记内容高度重叠，会出现 **AI Merge** 按钮：

1. 点击 AI Merge，AI 分析两篇笔记的内容差异
2. 自动生成一篇合并后的新笔记，保留两篇笔记中各自独有的信息
3. 两篇原文件不会被删除，而是以 `archived_` 前缀重命名归档（如 `archived_旧笔记.md`）

### AI Atom Upgrade（升级）

命令面板: `AI Atom Upgrade (Current Note)`

当你写了一篇原子笔记之后，Vault 中其他笔记可能包含与这个主题相关的新信息。Atom Upgrade 会：

1. AI 自动查找与当前原子相关的其他笔记
2. 从中提取当前原子尚未涵盖的新信息
3. 将新信息追加到原子笔记末尾，帮助你持续完善知识

## Vault-aware Chat — 知识感知聊天

### 打开方式

点击侧边栏的 🤖 图标，或命令面板输入 `Open Google Antigravity Chat`

### 它比普通聊天强在哪？

普通 AI 聊天只知道你当前发的消息。而 Vault-aware Chat 可以把你的 Vault 里的笔记一起带进对话，让 AI 回答时参考你自己的知识体系。

聊天里有两类上下文：

- **Smart Context**: 当前笔记、backlinks、最近打开的文件。它会在输入框上方预览出来，你发送前可以逐个删除。
- **Related Notes Context**: 根据你当前问题临时检索出的相关笔记（最多 3 篇）。

默认行为现在更保守：

1. **Smart** 和 **Related** 默认都不会自动注入
2. 点击聊天工具栏里的 **Smart** 按钮，才会把当前笔记、backlinks、recent 文件加入当前这条消息
3. 点击聊天工具栏里的 **Related** 按钮，才会把语义相关的笔记加入当前这条消息
4. 两类上下文都会先显示在输入框上方，你可以逐个删除再发送
5. 如果你更喜欢旧行为，可以到 设置 → Antigravity 中分别把 **Smart context** 或 **Related notes context** 切换成 **Automatic**

这样做的好处是：你能在发送前看见将要附带的上下文，也能决定这条消息到底要不要带入 Related Notes。

### 手动指定参考笔记

在消息中输入 `@[笔记路径]` 可以手动指定让 AI 参考某篇笔记，例如：

```
@[项目/技术选型.md] 帮我基于这篇笔记写一份总结
```

## 浮动工具栏 & Prompt 模板

> ⚠️ 需要登录 GitHub Copilot

### 浮动工具栏

在编辑器中**选中一段文字**后，会自动弹出一个浮动工具栏，提供以下 AI 快捷操作：

| 操作 | 说明 |
|------|------|
| 翻译 | 选择目标语言（支持 10 种语言），AI 将选中文字翻译为目标语言 |
| 润色 | 提供多种模式：改进写作 / 纠错 / 精简 / 扩写 / 简化 / 专业化 / 口语化 |
| 摘要 | 将选中的长段落生成一段简短摘要 |
| 解释 | AI 解释选中的概念或代码 |
| 续写 | 根据上下文继续往下写 |
| 行动项 | 从选中内容中提取出待办事项（to-do list） |

操作结果会直接显示在编辑器中，你可以选择接受或取消。

### 自定义 Prompt 模板

你可以创建自己的 AI 操作模板，在浮动工具栏中一键执行。

**设置方式**: 设置 → Antigravity → Prompt Templates → 添加新模板

模板中可以使用以下变量，运行时会被自动替换为实际内容：

#### 基础变量

| 变量 | 说明 |
|------|------|
| `{{selection}}` | 当前选中的文字 |
| `{{filename}}` | 当前文件名（如 `会议记录.md`） |
| `{{filepath}}` | 当前文件的完整路径 |
| `{{fulltext}}` | 当前文件的全部内容 |
| `{{frontmatter}}` | 当前文件开头 `---` 中的元数据内容 |
| `{{date}}` / `{{time}}` | 当前日期 / 当前时间 |

#### 交互变量（运行时弹出输入界面）

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{input:提示文字}}` | 运行时弹出输入框，让你填写内容 | `{{input:请输入目标受众}}` |
| `{{select:列表名}}` | 运行时弹出下拉选择框，从预定义列表中选择一项 | `{{select:语言}}` |
| `{{multiselect:列表名:N}}` | 运行时弹出多选框，最多选 N 项 | `{{multiselect:标签:3}}` |
| `{{random:列表名:N}}` | 从预定义列表中随机选取 N 项 | `{{random:风格:2}}` |

> 💡 `{{select:}}`、`{{multiselect:}}` 和 `{{random:}}` 中的"列表名"需要在模板设置中预先定义。例如定义一个名为"语言"的列表，包含 `[中文, English, 日本語, ...]`。

#### 高级变量

| 变量 | 说明 |
|------|------|
| `{{pins}}` | 你在聊天面板中"固定（Pin）"的所有笔记的内容。Pin 操作在聊天面板中完成 |
| `{{var:name}}` | 自定义变量，在模板设置中定义键值对（如 `name=张三`），运行时自动替换 |

#### 模板示例

```
请将以下内容改写为面向{{input:请输入目标受众}}的{{select:风格}}文章：

{{selection}}
```

## Inline Completion — 智能续写

### 什么是 Inline Completion？

Inline Completion（又称 Ghost Text）是编辑器中的实时续写功能——当你暂停输入时，光标后方会出现一段半透明的灰色建议文字。如果建议合适，按 **Tab** 接受；不满意就继续输入，建议会自动消失。

这和 VS Code 中 GitHub Copilot 的体验类似，但是在 Obsidian 编辑器中运行。

### 开启方式

设置 → Antigravity → **Inline Completion** 开关打开

### 支持的 AI 提供者

| 提供者 | 说明 | 适用场景 |
|--------|------|---------|
| **copilot** (默认) | 使用 GitHub Copilot API，需要已登录 | 最方便，不需要额外配置 |
| **openai** | 直接调用 OpenAI API，需要自行配置 API Key | 想用特定 OpenAI 模型 |
| **ollama** | 连接本地 Ollama 服务，完全离线 | 注重隐私、不想数据出本机 |

可在设置中的 **Completion Provider** 切换。

### 相关设置

| 设置 | 默认值 | 说明 |
|------|--------|------|
| Completion Model | `gpt-4o-mini` | 续写使用的模型。较小的模型响应更快，大模型质量更高 |
| Completion Delay | `500ms` | 停止输入多久后才触发续写请求。设太低会频繁请求，太高会感觉迟钝 |

### 使用技巧

- **写到句子一半时停顿** → 等待 0.5 秒 → 灰色续写出现 → Tab 接受
- **想要不同的建议** → 继续输入几个字改变上下文 → 会触发新的建议
- **在代码块内** → Inline Completion 同样工作，适合在笔记中写代码片段

## 设置项参考

| 设置 | 默认值 | 说明 |
|------|--------|------|
| Gateway URL | `http://localhost:3000` | Antigravity Gateway 服务地址。**如果你没有运行 Gateway，留空即可**，不影响其他功能 |
| Workspace | (自动检测) | 当前 Vault 的路径标识，一般不需要手动修改 |
| Smart context | `Manual` | 控制聊天里的当前笔记上下文是否自动参与消息。`Manual` 表示只有点击 **Smart** 按钮才会加入当前消息，`Automatic` 表示输入时自动加入 |
| Related notes context | `Manual` | 控制聊天里的 Related Notes 是否自动参与上下文。`Manual` 表示只有点击 **Related** 按钮才会加入当前消息，`Automatic` 表示输入时自动加入 |
| Log Level | `INFO` | 日志详细程度。遇到问题时可切换为 `DEBUG` 获取更多诊断信息 |
| Exclude Folders | `node_modules,.git,dist,build,.obsidian` | 这些文件夹中的文件不会被纳入知识索引。用逗号分隔多个文件夹名 |
| Inline Completion | 关闭 | 开启后，在编辑器中输入时会出现半透明的续写建议（类似 GitHub Copilot 的 Ghost Text），按 Tab 接受 |
| Completion Provider | `copilot` | 续写建议使用的 AI 服务 |
| Completion Model | `gpt-4o-mini` | 续写建议使用的模型。较小的模型响应更快 |
| Completion Delay | `500ms` | 停止输入多久后才显示续写建议，避免频繁触发 |
| Quick Action Provider | `copilot` | 浮动工具栏使用的 AI 服务 |
| Quick Action Model | `gpt-4o` | 浮动工具栏使用的模型。翻译/润色等操作使用这个模型 |

## 故障排查

### 索引不更新

- 命令面板输入 `Rebuild Knowledge Index` 可触发全量重建
- 检查笔记是否在排除目录（Exclude Folders）中——被排除的文件夹里的笔记不会被索引

### AI 功能不可用

- 打开设置页，检查 Copilot 是否显示"已登录"状态
- 确认你的 GitHub 账号有有效的 Copilot 订阅
- 查看详细日志: 命令面板输入 `Export Debug Logs`，日志会复制到剪贴板，可粘贴到文本文件中查看错误信息

### 性能问题

- **大型 Vault (>200 篇笔记)**: 首次打开时需要构建索引，可能需要几秒钟。之后只会增量更新修改过的笔记，不会重复全量扫描
- **网络图**: 自动限制最多显示 100 个节点，不会因笔记数量过多而卡顿
- **AI 增强**: 每次最多处理 20 篇笔记，请求间有 0.8 秒间隔。如果 Vault 中有大量未增强笔记，需要多次运行

### 断链修复不生效

- 确认目标文件存在且路径正确
- 检查是否有别名（`[[link|别名]]`）—  修复会保留别名
